## 1. 제품 개요

### 1.1 제품명

**META 학습심리정서검사 AI 에이전트 대시보드**

### 1.2 제품 요약

교사가 학생들의 학습심리정서검사(META) 결과를 다층적으로 분석하고, AI 기반 인사이트를 통해 맞춤형 코칭 전략을 수립할 수 있도록 지원하는 웹 애플리케이션입니다.

### 1.3 제품 비전

> "검사 결과 데이터를 교사가 즉시 이해하고 실천할 수 있는 코칭 전략으로 전환한다."

### 1.4 핵심 가치 제안

| 가치 | 설명 |
|------|------|
| **직관적 시각화** | 38개 요인, 11개 중분류, 5대 영역을 계층적 대시보드로 시각화 |
| **AI 기반 해석** | Gemini AI가 검사 결과를 교사 언어로 번역하여 3줄 요약, 코칭 제안 제공 |
| **다층 분석** | 교사 전체 → 학급 → 학생 순으로 드릴다운하며 점진적 인사이트 획득 |
| **업무 통합** | 생활기록부 문구 생성, 상담일정 관리, 관찰 메모를 하나의 플랫폼에서 처리 |
| **개인정보 보호** | AI 호출 시 자동 PII 마스킹으로 학생 개인정보 유출 방지 |

## 2. 대상 사용자

### 2.1 주요 사용자: 담임교사

| 항목 | 설명 |
|------|------|
| **역할** | 초등/중등학교 담임교사 |
| **목표** | 학생 검사 결과를 빠르게 파악하고, 근거 기반 코칭 전략 수립 |
| **페인 포인트** | 38개 요인의 복잡한 T점수를 해석하기 어려움, 생활기록부 작성 부담, 상담 관리 분산 |
| **기대 효과** | 데이터 해석 시간 80% 단축, 생활기록부 초안 자동 생성, 상담 통합 관리 |

### 2.2 사용 시나리오

```
[시나리오 1] 학기초 학급 진단
교사 → L1 대시보드에서 담당 반 전체 현황 확인
    → L2 학급 대시보드에서 유형 분포, 관심 필요 학생 파악
    → L2.5 상세 분석에서 학급 강점/약점 기반 운영 전략 수립

[시나리오 2] 개별 학생 심층 분석
교사 → L2 학급 대시보드에서 특정 학생 클릭
    → L3 학생 대시보드에서 AI 총평, 유형 분석, 특이점 확인
    → 데이터 해석 도우미로 궁금한 점 질문
    → 우측 패널에서 생활기록부 문구 생성 / 상담 기록 작성

[시나리오 3] AI 심층 상담
교사 → AI Room에서 특정 학생 또는 학급 선택
    → 대화형으로 검사 결과 해석, 코칭 방법, 학급 운영 조언 획득

[시나리오 4] 상담 일정 관리
교사 → 스케줄 페이지에서 주간/월간 상담 일정 관리
    → 상담 완료 후 기록 작성 → L3 학생 대시보드와 동기화
```

## 3. 도메인 지식

### 3.1 META 검사 구조

```
38개 세부 요인 (T점수: 20~80, 평균 50, 표준편차 10)
  ↓ 그룹핑
11개 중분류
  ↓ 그룹핑
5대 영역
  ├── 자아강점 (정적)
  ├── 학습디딤돌 (정적)
  ├── 학습걸림돌 (부적)
  ├── 긍정적공부마음 (정적)
  └── 부정적공부마음 (부적)
```

### 3.2 LPA 유형 분류

학생의 38개 T점수 패턴을 Latent Profile Analysis 알고리즘으로 3개 유형으로 분류합니다.

**초등학교**

| 유형 | 비율 | 특징 |
|------|------|------|
| 자원소진형 | 30.55% | 심리자원 낮음, 스트레스 높음 |
| 안전균형형 | 35.47% | 전반적 균형, 점검능력 약함 |
| 몰입자원풍부형 | 33.98% | 동기 높음, 시험전략 보완 필요 |

**중등학교**

| 유형 | 비율 | 특징 |
|------|------|------|
| 무기력형 | 35.4% | 동기 저하, 무력감, 목표 설정 어려움 |
| 정서조절취약형 | 38.0% | 스트레스 관리 미흡, 감정 기복, 불안 경향 |
| 자기주도몰입형 | 26.6% | 자율적 학습, 높은 성취동기, 효과적 시간관리 |

### 3.3 관심 필요 학생 판별 기준

| 요인 유형 | 기준 | 의미 |
|-----------|------|------|
| 정적 요인 (isPositive=true) | T점수 ≤ 39 | 긍정 자원이 현저히 부족 |
| 부적 요인 (isPositive=false) | T점수 ≥ 60 | 부정 요인이 현저히 높음 |

## 5. AI 기능 명세

### 5.1 AI 서비스 아키텍처

```
컴포넌트
  → callAI({ feature, messages })
    → ai.ts (Provider 선택, 시스템 프롬프트 적용)
      → aiPrompts.ts (기능별 프롬프트 조회)
      → gemini.ts (API 호출, PII 마스킹, 재시도)
        → Gemini 2.5 Flash API (v1beta)
```

### 5.2 AI 기능 목록

| Feature ID | 기능명 | 사용처 | 설명 |
|------------|--------|--------|------|
| `analysis` | 학생 AI 분석 총평 | L3 학생 대시보드 | 11개 중분류 결과를 3문장(강점/약점/코칭)으로 요약 |
| `record` | 생활기록부 문구 생성 | L3 우측 패널 | 검사 결과 기반 생활기록부 기재 문구 자동 생성 (5개 카테고리) |
| `dataHelper` | 데이터 해석 도우미 | L3 플로팅 챗봇 | 7개 사전 질문에 대한 맞춤형 AI 해석 제공 |
| `assistant` | AI 어시스턴트 | AI Room | 교사와의 멀티턴 대화로 검사 결과 심층 분석 |
| `classAnalysis` | 학급 분석 총평 | L2.5 상세 분석 | 학급 전체 특성 요약(overall) + 핵심 포인트(keyPoint) |

### 5.3 AI 안전장치

| 장치 | 설명 |
|------|------|
| **PII 자동 마스킹** | 이름, 학번, 생년월일, 학교명을 AI 호출 전 자동 마스킹 |
| **학생 별칭 시스템** | AI Room에서 student_A, student_B 등으로 치환, UI에서 역변환 |
| **Placeholder 금지** | 모든 프롬프트에 `[학생]`, `[학교명]` 등 placeholder 사용 금지 규칙 포함 |
| **429 자동 재시도** | Rate limit 시 최대 3회 지수 백오프 재시도 |

## 부록 A: 용어집

| 용어 | 설명 |
|------|------|
| **META** | 학습심리정서검사 (제품명) |

| **정적 요인** | 점수가 높을수록 긍정적인 요인 (자아강점, 학습디딤돌 등) |
| **부적 요인** | 점수가 높을수록 부정적인 요인 (학습걸림돌, 부정적공부마음 등) |
| **중분류** | 38개 요인을 묶은 11개 상위 카테고리 |
| **5대 영역** | 자아강점, 학습디딤돌, 학습걸림돌, 긍정적공부마음, 부정적공부마음 |
| **PII** | Personally Identifiable Information (개인식별정보) |
| **RAG** | Retrieval-Augmented Generation (검색 증강 생성) |
| **Merit Score** | 학급 프로필 산출 시 사용하는 영역별 점수화 기법 |
