## 4. 문항 순서 (셔플링)

### 4.1 셔플링 정책
- 문항은 **셔플링이 적용된 순서**로 제시
- 같은 변인의 문항이 연속으로 나오지 않도록 배치
- 검사 시 제시되는 순서와 원래 문항번호는 다름

## 5. 문항 점수 처리

### 5.1 기본 점수 계산
- 각 문항: 1~5점
- 변인 점수 = 해당 변인 문항 합계 / 문항 수 (평균)

### 5.2 역채점 문항
- 현재 검사에서 역채점 문항 없음
- 모든 문항 동일 방향으로 채점

### 5.3 변인 점수 → T점수 변환
```
변인 T점수 = {(변인평균 - 기준평균) / 기준SD} × 10 + 50
```

## 6. API 데이터 모델

### 6.1 문항 데이터 구조 (TypeScript)

```typescript
interface QuestionItem {
  questionId: number;           // 원래 문항번호
  displayOrder: number;         // 제시 순서 (셔플링 적용)
  depth1: string;               // 대분류
  depth2: string;               // 중분류
  depth3: string;               // 변인명
  content: string;              // 문항 내용
  factorType: 'positive' | 'negative';  // 정적/부적
  isReversed: boolean;          // 역채점 여부
}
```

### 6.2 응답 데이터 구조

```typescript
interface QuestionResponse {
  questionId: number;
  displayOrder: number;
  response: 1 | 2 | 3 | 4 | 5;   // 응답값
  responseTime?: number;         // 응답 소요시간 (ms)
}
```

### 6.3 변인 점수 계산

```typescript
interface VariableScore {
  variableName: string;          // 변인명 (Depth 3)
  rawMean: number;               // 원점수 평균
  tScore: number;                // T점수
  percentile: number;            // 백분위
  level: string;                 // 등급
  questionIds: number[];         // 해당 문항 ID 목록
}
```
